Sass란?

Sass는 CSS 전처리기입니다.
Sass는 CSS의 반복을 줄여서 시간을 절약해줍니다.

Sass의 특징

  • Sass는 Syntactically Awesome Stylesheet의 약자입니다.
    (해석하자면 구문론적으로 멋진 스타일시트!^^)
  • Sass는 CSS의 확장입니다.
  • Sass는 CSS 전처리기입니다.
  • Sass는 모든 버전의 CSS와 완벽하게 호환됩니다.
  • Sass는 CSS의 반복을 줄여 시간을 절약합니다.
  • Sass는 Hampton Catlin이 디자인하고 2006년 Natalie Weizenbaum이 개발했습니다.
  • Sass는 무료로 다운로드하여 사용할 수 있습니다.

왜 Sass를 사용하는가?

스타일시트는 점점 더 커지고, 더 복잡해지고, 유지 관리가 더 어려워지고
있습니다. CSS 전처리기가 도움이 될 수 있는 곳입니다.
Sass를 사용하면 변수, 중첩 규칙, 믹스인, 가져오기, 상속, 내장 함수 및
기타 항목과 같이 CSS에 존재하지 않는 기능을 사용할 수 있습니다.

Sass는 어떻게 작동하는가?

브라우저는 Sass코드를 해석하지 못합니다.
따라서 사스로 작성한코드는
반드시 전처리기로 일반 CSS 코드 변환이 필요합니다.
지금은 VS Code 툴에서
아래 두가지 툴을 설치하면 간단히 사용할 수 있습니다.

  1. Sass 라이브 컴파일러 : Live Sass Compiler
  2. Sass 포멧터 : Sass Formatter -> Prettier있으면 패스!

Sass 라이브 컴파일러를 설치후 VS Code툴 하단라인에
"Watch Sass"를 클릭하면 "Working on it..." 으로
작동 전환표시 후 "Watching..." 으로 변경된다.
이 상태가 실시간으로 작성된 Sass파일을
같은이름의 CSS일반 파일로 변환해 주는 것이다!
