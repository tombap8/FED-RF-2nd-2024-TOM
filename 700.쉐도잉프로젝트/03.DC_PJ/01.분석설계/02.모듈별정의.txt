[ 모듈별 정의 : 구조정의/객체화/기능정의 ]

[1] 배너모듈
    - 모듈명: Banner (JS/CSS)
    (1) 구조정의 : 
        Root > div.banner 
        > 
        (ul.slider 
            > li 
            > img 
            + section.bantit 
            > h2 + p + button) 
        
        + button.abtn 
        + ol.indic
        
    (2) 객체화 : data/banner.js
        {
            "src": 배너이미지경로
            "tit1": 작은타이틀
            "tit2": 큰타이틀
            "cont": 배너 소개내용
            "btn": 컨텐츠 이동버튼
        }
    (3) 기능정의 :
        - 배너가 하나일때는 이동버튼 및 블릿 안나옴
        - 배너가 여러개이면 이동버튼은 양쪽에 나오고
        블릿은 배너 개수만큼 생성하여 현재 배너를 표시함
        - 배너가 한 페이지에 동시에 여러개를 보여주도록
        상대적인 기능으로 구현함
        (메인 페이지에 배너가 실제로 2개 배치됨!)

[2] 섹션소개모듈
    - 모듈명: SecIntro (JS/CSS)
    (1) 구조정의 :
        Root > section > img Box + title Box + button Box
    (2) 객체화 :
        {
            "isrc":"이미지경로",
            "tit":"작은타이틀^큰타이틀",
            "btn":"버튼문구",
            "link":"이동카테고리",
        }
        -> "tit" 속성은 ^로 데이터를 구분함
        -> 컴포넌트별 객체화 데이터 반영은 기본
    (3) 기능정의 : 
        1) 버튼 오버시 하일라이트
            (mouseover/mouseout) + css
        2) 버튼 클릭시 페이지이동
            (click) + location(라우트이동)
 
[3] 비디오 소개 모듈
    - 모듈명: VidIntro (JS/CSS/data)
    (1) 구조정의 :
        Root > video box(>video+img) 
                + title box(>heading+p tag)
    (2) 객체화 :
        {
            "vsrc":"유튜브동영상경로",
            "stit":"작은타이틀",
            "btit":"큰타이틀",
            "sum":"요약소개",
            "desc":"설명",
            "link":"관련링크",
        }
        
        [ 데이터조건 ]
        link 속성 값 셋팅시 하나의 링크만 셋팅할것!
        링크가 있을 경우 sum 또는 desc 속성값에
        단 하나의 *표시만 어디든지 넣는다!

    (3) 기능정의 : 
        1) 동영상영역 클릭시 동영상 재생/멈춤
        2) 관련링크 새창열기

[4] 비디오스와이프모듈
    - 모듈명: VidSwipe (JS/CSS/data)

    (1) 구조정의 :
        스와이퍼모듈 + 비디오재생모듈
        Root>
        section.vidswbox >
            ( h2.tit + <SwiperVid />) +
            ( section.vidbx > 
                div.playvid >
                    h2.ifrtit + iframe + button )

        ->> [ SwiperVid 플러그인 모듈 구조 ]
        Root>
        div.swiper-slide.swiper-slide-active
            > section.sw-inbox
                > (div.vid-img > img + 폰트어썸 아이콘)
                + (div.vid-tit > h4 + h3)
                    
                

    (2) 객체화 :
        1) 비디오 스와이프 모듈 정보 : data/vid_swipe.js
        (catTit)    
        {
            카테고리명: 타이틀정보
        }

        2) 스와이퍼 리스트 정보 : data/swiper_vid.js
        {
            isrc:"",
            vsrc:"",
            cat:"",
            tit:"",
        }
        1) 이미지경로 : isrc
        2) 유튜브URL : vsrc
        3) 카테고리 : cat
        4) 타이틀 : tit

    (3) 기능정의 : 
        1) 비디오 메뉴 스와이프 기능(스와이퍼)
        2) 비디오 메뉴 클릭시 별창 동영상 재생기능(닫기버튼)

[5] 캐릭터모듈
    - 모듈명: CatList (JS/CSS/data)

    (1) 구조정의 :
        제목 + 스와이퍼모듈
        -> 하위 스와이퍼모듈(이미지+이름)

    Root >
        section.cat-swbox >
            h2.tit + 스와이퍼 컴포넌트(<SwiperCat />)

            -> 스와이퍼 컴포넌트 구조:
                스와이퍼 리스트 >
                    a링크(<Link>) >
                        section.sw-inbox2 > img
                        +
                        div.cat-tit2 > h3
        
    (2) 객체화 :
        {
            idx : "",
            tmsrc:"",
            cname:"",
            cdesc:"",
            facts:"",
            alignment:"",
            type:""
        }
        1) 인덱스번호 : idx (유일키==기본키)
        2) 썸네일 이미지경로 : tmsrc
        3) 캐릭터이름 : cname
        4) 캐릭터설명 : cdesc
        5) 캐릭터명세 : facts
        6) 캐릭터분류 : alignment(히어로/빌런/복잡)
        7) 캐릭터타입 : type(개인/단체)

    (3) 기능정의 : 
        1) 캐릭터 메뉴 스와이프 기능(스와이퍼)
        2) 캐릭터 메뉴 클릭시 캐릭터 상세페이지 이동(라우터사용)
